Typescript



“TypeScript is a typed superset of JavaScript that compiles to plain JavaScript. Any browser. Any host. Any OS. Open source.”

-Offical TypeScript Website

What is TypeScript?


  • TypeScript was made by and maintained by Microsoft.
  • It is a superset of JavaScript.
    • This means that all JavaScript code is valid TypeScript code.
  • Uses extension .ts

Why is it Useful?



  • It is used in many large applications and can be scalable.
  • You can choose between statically typed and dynamically typed.
    • Statically typed allows for types of variables are known at compile time.
    • It still is possible to use JavaScript’s Dynamic typing.
    • You get to choose how aggressive you want your variable declaration to be.
  • It has many extra features and types that may be useful in niche situations.
    • It can be compiled back into JavaScript because it is a superset of JavaScript.

Classes in TypeScript


In TypeScript, classes were implemented to allow for object-oriented programming, if the programmer so wishes. Object-oriented programming allows for parts of the code to be completed and then combined at the end to make a cohesive product. TypeScript also allows for inheritance and constructors.

This is why TypeScript claims to be scalable and good for large applications. Most people understand how this form of programming is useful for a larger group of programmers and for applications. JavaScript may be good for smaller applications, but gets messy for larger and more industrial uses.

Interfaces


“One of TypeScript’s core principles is that type-checking focuses on the shape that values have. This is sometimes called “duck typing” or “structural subtyping”. In TypeScript, interfaces fill the role of naming these types, and are a powerful way of defining contracts within your code as well as contracts with code outside of your project.”
-Official TypeScript website

Interfaces are useful for roughly defining abstract objects. This may be useful in applications like parameters. Instead of passing multiple parameters, you can pass an interface and not have to deal with many inputs.


Modules/Namespaces


Namespaces are called “internal modules” and modules themselves represent “external modules.” Both of these are useful for encapsulation and modularity. This allows for easier organization and such.

Modules are preferred as these contain files or lines of code within their own environments so that they won’t interfere with one another in the global scope.

More Variable Types


There are many new types that are added with TypeScript. These include:
  • Enum
    • A collection of named constants
  • Symbol
    • Immutable, unique, and can act like keys for object properties
  • Any
    • Key word that means unknown variable that, unlike object, allows for arbitrary methods to be applied
  • Never
    • Key word that ensures a variable or function does not have or return a value
  • And many more!


This allows for a variety of types that can be used for many specific situations.
